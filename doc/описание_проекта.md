# Чек-лист по проекту

**ФИО:** Дубровских Никита Евгеньевич, группа 221-361

**Название проекта:** Разработка информационной системы для создания туристических маршрутов с геопривязкой фотографий и интерактивным обменом на основе микросервисной архитектуры

---

## Какую проблему решает проект?

Гиды, инструкторы и организаторы походов фотографируют ключевые точки маршрутов (достопримечательности, опасные участки, живописные виды, места стоянок), но не имеют удобного инструмента для размещения этих фотографий на интерактивной карте с привязкой к конкретным точкам маршрута и последующим обменом результатом по ссылке. Существующие решения либо привязывают фото автоматически по GPS-метке без ручного контроля (Wikiloc), либо не строят маршрут между фототочками (Google My Maps), либо вообще не предназначены для создания маршрутов с фотографиями (Strava).

---

## У кого возникает проблема? Анализируем целевую аудиторию.

1. **Гиды и инструкторы** — основная ЦА. Создают маршруты с фотографиями для демонстрации клиентам: что ждёт участников похода, какие виды они увидят, какие сложные участки пройдут. Хотят поделиться интерактивной картой по ссылке.
2. **Организаторы походов и турклубы** — делятся маршрутами с участниками группы до и после похода. Собирают обратную связь через комментарии и рейтинги.
3. **Активные путешественники** — документируют маршруты с фотографиями для личного использования и обмена опытом с единомышленниками.
4. **Туристические компании** — используют платформу как витрину маршрутов с фотографиями для привлечения клиентов.

---

## Каким образом проблема решается в настоящее время?

- Фотографии отправляются в мессенджеры и социальные сети без привязки к карте — теряется географический контекст.
- Маршрут рисуется в Google My Maps (можно поставить пины с фото, но нельзя построить маршрут между ними, нет социальных функций).
- Используется Wikiloc (фото привязываются автоматически по GPS-метке времени, без точного ручного контроля; нет интерактивного шаринга с фото-попапами; платная подписка для полного функционала).
- Составляются PDF-отчёты с вставленными фотографиями и скриншотами карт — не интерактивно, не масштабируемо.

---

## Какие альтернативы? Аналоги и конкуренты.

1. **Wikiloc** — крупнейший сервис туристических треков. Фото привязываются к GPS-треку автоматически по временной метке, без ручного размещения на конкретных точках. Шаринг не даёт интерактивную карту с фото-попапами. Freemium: навигация и оффлайн платные. Нет модерации. Закрытый код.
2. **Google My Maps** — можно ставить пины с фотографиями, но нет построения маршрута между ними. Нет социальных функций (комментарии, лайки, рейтинги). Нет серверной обработки фото (компрессия, миниатюры).
3. **Strava** — запись спортивной активности, а не создание маршрутов с фототочками. Нет ручного создания маршрута с привязкой фото. Нет шаринга интерактивной карты.
4. **Komoot** — платные региональные карты. Фотографии второстепенны. Нет интерактивного шаринга с попапами фотографий.
5. **AllTrails** — платная подписка для полного функционала. Слабое покрытие территории РФ. Фото не привязаны к конкретным точкам маршрута на карте.

## Какая исследовательская часть будет в проекте?

1. Анализ рынка туристических приложений и сервисов для создания маршрутов с фотографиями.
2. Исследование потребностей целевой аудитории (гидов, организаторов походов, путешественников).
3. Сравнительный анализ архитектурных подходов: микросервисная архитектура vs монолит — обоснование выбора для данной предметной области.
4. Обзор алгоритмов: формула Гаверсинуса для расчёта расстояний по координатам, формула Нейсмита для оценки времени пешего маршрута, алгоритмы компрессии изображений.
5. Сравнительный анализ технологий: языки программирования (Rust, Go, TypeScript), СУБД (PostgreSQL vs MongoDB), брокеры сообщений (NATS vs RabbitMQ vs Kafka).

## Какие данные будут использованы?

- **OpenStreetMap** — картографические тайлы (лицензия ODbL, открытые данные).
- **Open-Meteo API** — данные о высоте рельефа (открытый API, бесплатный).

- Работа с персональными данными минимальна (email). Фотографии загружаются самими пользователями.

## Актуальность работы

Существующие сервисы (Wikiloc, Google My Maps, Strava, Komoot, AllTrails) не позволяют удобно решить задачу гидов и организаторов: ручная привязка фотографий к произвольным точкам маршрута на интерактивной карте с последующим шарингом по ссылке. Wikiloc привязывает фото по GPS автоматически без ручного контроля; Google My Maps не строит маршруты между точками; Strava не про создание маршрутов с фотографиями. Guide Helper заполняет эту нишу: размещение фото на произвольных точках карты, построение маршрута, шаринг интерактивной карты по ссылке, социальные функции, серверная обработка фотографий — бесплатно и с открытым кодом.

---

## Цель работы

Разработать кроссплатформенную информационную систему для создания туристических маршрутов с привязкой фотографий к точкам на интерактивной карте и возможностью совместного использования на основе микросервисной архитектуры.

## Задачи

1. Проанализировать существующие решения для создания и обмена туристическими маршрутами с фотографиями и выявить их недостатки.
2. Проанализировать целевую аудиторию и определить функциональные требования к системе.
3. Спроектировать микросервисную архитектуру системы с обоснованием выбора технологий.
4. Спроектировать реляционную базу данных и API взаимодействия компонентов.
5. Разработать серверную часть: микросервисы аутентификации, управления маршрутами, кэширования карт и асинхронной обработки фотографий.
6. Разработать клиентскую часть на React с интеграцией интерактивных карт (Leaflet).
7. Реализовать систему ролей (администратор, модератор, пользователь) и модерации контента.
8. Реализовать PWA-версию с поддержкой оффлайн-режима и десктоп-приложение (Tauri).
9. Организовать CI/CD, контейнеризацию и оркестрацию (Docker, Kubernetes).
10. Провести тестирование и анализ производительности системы.
